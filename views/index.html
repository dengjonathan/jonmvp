hash table

A hash table stores and retrieves key/ value pairs at constant time. Try it out!

Explanation

A hash table is a data structure that supports storage and retrieval of values with an average of O(1) time complexity, or constant time, meaning that in most cases the amount of time to store or retrieve a value does not increase as the data set grows. However, as the hash table becomes too full (over 75% of possible indices are filled), storage and retrieval will become inefficient due to collisions. Once, the utilization surpasses 75%, the hash table will double in size and reassign all values stored. While this operation is expensive, it should be rather rare and amortized over all other storage/retrieval transactions. Additionally, to save space, the hash table will halve in size when its utilization drops below 25%.
